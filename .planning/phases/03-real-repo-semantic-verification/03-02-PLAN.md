|----------|-------------|
| manifest contract → proof harness | Committed manifest data controls which repos and labels enter approval evidence |
| wrapper artifacts → parity summaries | Generated comparison output must not replace or misrepresent raw evidence |
| sequential repo run → parallel repo run | Missing or mismatched mode pairs could create false parity claims |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-03-05 | Repudiation | run-level parity reporting | mitigate | Persist `requested_mode`, `actual_mode`, and `semantic_pairs` so every comparison can be traced back to concrete wrapper paths. |
