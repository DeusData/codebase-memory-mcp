## Placement rules

1. Put product code in the narrowest existing `src` module.
2. Put extraction and language work in `internal/cbm`; keep third-party grammar
   sources below its `vendored` boundary.
3. Put required C suites directly in `tests` and register them in `Makefile.cbm`
   and `tests/test_main.c`. Put open-bug reproduction cases in `tests/repro`.
4. Put React code only below `graph-ui/src`. Build output and TypeScript build
   metadata are never source files.
5. Put package-manager metadata under its matching `pkg` shelf and update it
   through `scripts/sync-version.py`.
6. Regenerate this index and run `make organization` after adding a shelf or module.
